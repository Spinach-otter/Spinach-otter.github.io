Hexo+Github发布个人博客

hexo框架+GitHub实现静态博客

hexo安装

Node.js安装

因为hexo是基于node.js的框架，所以先下载安装好node.js环境。

node.js官网下载长期版本

使用下列shell命令检测node.js是否安装成功
node -v
terminal终端返回： v18.13.0 类似的版本号

hexo安装使用

hexo框架中需要下载一些依赖包，但是npm镜像国外的所以下载超级慢。所以先把npm的镜像设置为淘宝的

注意：如果使用npm时，一直有error说明权限不够。解决办法为在所有的npm命令前面加上sudo，变成超级管理

清空代理
npm config set proxy null
如果报错就换成超级管理身份
sudo npm config set proxy null

清空缓存
npm cache clean --force

重新设置镜像
npm config set registry https://registry.npm.taobao.org

用cd命令进入你要安装hexo的目录路径

例如：
cd /User/Spinach/Desktop/website/bolg

安装hexo指令

sudo npm install hexo-cli -g

初始化hexo博客框架

hexo init blog

开始运行hexo

使用指令先安装依赖
npm install

在本地运行hexo
hexo start
返回：hexo is running at http://localhost:4000
打开任意浏览器搜索http://localhost:4000 即可看到hexo在本地发布的博客框架内容

创建一篇新的博客

hexo n "我的第一篇博客"
返回：Created: /User/Spinach/Desktop/website/blog/source/_post/我的第一篇博客.md

使用markdown语法把文件 “我的第一篇博客.md” 编辑好后，退回到安装hexo的目录重新发布

退回安装了hexo的目录
cd /User/Spinach/Desktop/website/blog
清理一下
hexo clean
生成一下
hexo g
重新运行一下
hexo s
浏览器打开4000端口即可看到 “我的第一篇博客.md” 的内容

以上只是在本地发布了博客内容，没有发布在互联网上，可以借助GitHub pages为我们提供一个互联网访问入口

Git安装及链接

Git安装

Mac自带git工具，不需要额外下载

Win下载git地址

使用指令查看git是否下载成功
git --version
返回： git version 2.37.1(Apple Git-137.1) 类似的版本信息

SSH配置

需要使用SSH来链接本地与GitHub

所以先得到本地的SSH公钥

ssh-keygen -t rsa -C "2947387166@qq.com" 
上面邮箱换成自己注册Github的邮箱

一路回车即可，如果提示是否重新生成，直接填y

进入ssh目录下
cd .ssh
查看公钥
cat id_rsa.pub
把显示出的公钥信息复制好

去Github填写本地的公钥，配置SSH

进入github.com登入
点击头像，选择settings进入
点击SSH and GPG keys，然后点击New SSH Key
Title框填自己GitHub上的用户名，key type默认，key将刚刚复制的ssh公钥黏贴进去，然后Add SSH Key

将hexo发布在GitHub仓库里

  1. 在GitHub页面new一个repository，Repository name填写username.github.io；username是自己的GitHub用户名。然后直接Create Repository

  2. 将hexo界面发布到GitHub上。

    打开blog文件夹下的_config.yml，修改以下三个地方：

    一：title改成自己想要的文字，language改成zh-CN，author改成自己想要的发布者的昵称

    二：16行左右的url改成https://username.github.io ；username是GitHub的用户名

    三：文件最后改成下面这样，repo用的格式git@github.com:username/username.github.io.git

    修改完保存后退出

  3. 安装hexo中的git插件

    npm install hexo-deployer-git --save

  4. 使用指令配置git的邮箱和用户名

    git config --global user.email "2947387166@qq.com"
git config --global user.name "Spinach"
邮箱和用户名都是自己GitHub上的

  5. 将本地的hexo发布到GitHub上

    hexo d

接下来就可以通过GitHub的这个仓库对应的浏览器链接访问自己的博客啦！

美化DIY

主题替换，在https://hexo.io/themes/ 找到自己喜欢的博客主题进行安装。我自己用的是matery。

把下载好的主题文件包，例如“matery”

放到安装hexo的目录下，比如“/User/Spinach/Desktop/website/bolg”

修改/User/Spinach/Desktop/website/bolg/_config.yml文件中theme: 下载的主题文件包名字

保存修改退出后，重新发布hexo即可

运行
hexo clean
hexo g
hexo d
